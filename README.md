# Im7o (إيمحو) - The Ultimate Party Games Platform 𓋹

منصة **Im7o** هي منصة ألعاب جماعية وحفلات (Party Games) مبنية على نظام أوبن سورس يتيح لأي شخص ابتكار وإضافة ألعابه الخاصة.
تعتمد المنصة على معمارية مبتكرة حيث تعمل كـ **Custom Git-Backed CMS**: أي لعبة يتم إنشاؤها عبر لوحة تحكم المنصة تتحول تلقائياً إلى Pull Request مسودة على Github، ويتم إعلام الصانع عبر البريد الإلكتروني. بمجرد قيام الإدارة بمراجعة الكود والموافقة (Merge)، يُرسل بريد القبول وتصبح اللعبة متاحة فوراً لجميع اللاعبين!

## الميزات الرئيسية 🌟

- **العمل دون اتصال (PWA Offline Mode)**: التطبيق يعمل كتطبيق ويب تقدمي (PWA)، ما يعني إمكانية اللعب في الأماكن المغلقة التي لا يوجد بها إنترنت بكل سلاسة، مع تحديث محتويات الألعاب تلقائياً في الخلفية.
- **نظام ثنائي اللغة (Bilingual i18n)**: واجهة المنصة تدعم اللغتين العربية والإنجليزية بتبديل لحظي (Runtime Switching)، لتوسيع قاعدة اللاعبين وصناع المحتوى.
- **تصميم فرعوني ديناميكي (Egyptian Aesthetic UI)**: واجهات أنيقة بتأثيرات بصرية راقية مستوحاة من الآثار المصرية مع لمسات ذهبية ومؤثرات حركية (Micro-animations) تضفي حيوية على تجربة اللعب.
- **صفحة هبوط تفاعلية (Landing Page)**: صفحة ترحيبية تشرح طريقة عمل المنصة مع توجيه سلس للبدء في اللعب أو قراءة المصدر.
- **محركات ألعاب متعددة**: المنصة تمتلك نظام **Engines** متقدم يتيح ابتكار ميكانيكيات لعب مختلفة:
  - `Classic`: كلاسيكي بالدور المباشر.
  - `MCQ / Trivia`: سؤال وجواب (يخفي الإجابة حتى يقرر المضيف إظهارها).
  - `Taboo`: الكلمات الممنوعة (يعرض الكلمة المراد تخمينها مع قائمة بالكلمات الممنوع نطقها).
  - `Hot Potato`: البطاطس الساخنة (تمرير الهاتف مع عداد قنبلة متسارع).
  - `Imposter / Spyfall`: الجاسوس (تمرير الهاتف لرؤية الأدوار السرية ثم الاستجواب).
- **إدارة المحتوى المؤتمتة عبر Github (Automated CMS)**: يقوم الباك إند برفع الإضافات مباشرة كطلبات سحب (Pull Requests) لحظة تقديمها، مما يمنع ضياع أي بيانات.
- **نظام سير العمل والتنبيهات (Workflow & Emails)**: 
  - عندما يرسل الصانع لعبة جديدة عبر الواجهة، يُنشئ النظام تلقائياً Pull Request ويُرسل بريداً إلكترونياً للصانع (يطلب منه الانتظار).
  - يمكن للإدارة فحص اللعبة ودمجها (Merge) والموافقة عليها.
  - عند الموافقة، يُرسل النظام بريداً بالقبول للصانع مع تصميم فرعوني مميز.

## 🛠️ كيف تعمل المنصة؟ (Architecture)

يتكون المشروع من شقين:
1. **Frontend (Next.js)**: واجهة المستخدم الأساسية، تتضمن شاشات اللعب، موجه المحركات، PWA، ولوحات التحكم. تعتمد في قراءة الألعاب على ملفات الـ JSON في `src/data/games/`.
2. **Backend (FastAPI)**: خدمة خفيفة ومستقلة لإدارة عمليات الـ OAuth، واستقبال طلبات لوحة التحكم، والتواصل التلقائي مع واجهة Github API لإنشاء الـ Pull Requests في الخلفية، وإرسال تنبيهات البريد الإلكتروني.

## 🚀 تشغيل المشروع محلياً (Local Development)

### متطلبات التشغيل
- Node.js (لـ Frontend)
- Python (لـ Backend)
- حساب Github و Personal Access Token لرفع الـ Pull Requests.
- إعداد سيرفر SMTP لإرسال رسائل البريد الإلكتروني (مطلوب للـ Backend).

### خطوات التشغيل
1. قم باستنساخ المشروع وتثبيت الحزم للواجهة الأمامية (`npm install`).
2. قم بتشغيل الواجهة الأمامية عبر `npm run dev`.
3. انتقل لمجلد الباك إند وثبّت حزم البايثون، ثم قم بتشغيله عبر Uvicorn (`uvicorn main:app --reload`).
4. تأكد من إعداد ملف `.env` في الباك إند ليتضمن الـ `GITHUB_TOKEN`، `SMTP_SERVER`، وباقي الإعدادات.

### استكشاف الأخطاء وإصلاحها (Troubleshooting)
- **مشكلة تسجيل الدخول بـ Google OAuth (يعود إلى صفحة الهبوط)**: 
  إذا كنت تواجه مشكلة `ExpiredTokenError` بسبب اختلاف توقيت جهازك المحلي عن خوادم جوجل، أو خطأ `InsecureTransportError` عند استخدام IP بدلاً من localhost:
  - تأكد من إضافة `os.environ['AUTHLIB_INSECURE_TRANSPORT'] = '1'` في الباك إند (تمت إضافتها بالفعل في `main.py`).
  - تأكد أن تسجيل الدخول يتم حصراً عبر النطاق `localhost` وليس `127.0.0.1`. الباك إند معدّ ليتجاهل فحص وقت JWT (`exp`) باستخدام الاستدعاء المباشر لمعلومات المستخدم `userinfo` لتفادي أخطاء فرق التوقيت (Clock Skew).
